## Goal
The four `@pytest.mark.backtest_regression` snapshots (S27/S32/S34/S35) fail on clean
`origin/main` (`0a9c17c`). They run in a ~7h slow lane deselected from fast PR CI, so the
drift accumulated silently. Determine *why* each drifted (engine regression vs benign
merged fix vs harness noise), then re-baseline deliberately. **RA-5** tag (maintenance;
NOT a DECISIONS D-number — D-series live at D23). Orthogonal to the deferred D19/D21
re-baseline (#320 scope doc), which is a *future intentional* EV-output change.

## What we tried
1. **Determinism gate first (load-bearing).** If the harness were non-deterministic, the
   sub-1% drifts would be run-to-run noise and re-baselining unsafe.
2. **Revert-from-main isolation** for each named suspect (D20 `6087d51`, E4-E6 `537ece1`,
   guards `90a9362`), holding everything else at main — cleaner than a bisect through the
   D19/D21 apply/revert interleave.
3. **Pin-vintage resolution** from each snapshot JSON's *commit history* + `git merge-base`
   (NOT the `generated_at` field, which is a placeholder in s27).
4. **Pin-SHA reproduction test** for the residual (run s27 at its own pin SHA).

## What worked — determinism GATE passed
- Full **s35 ×2** on one clean SHA → metric sections **byte-for-byte identical** (only the
  excluded `generated_at` differs). 3-process fast probe (OS-randomized PYTHONHASHSEED)
  byte-identical (`ev_dollars_sha=5011a32dfa20e7df`). By construction: block-bootstrap /
  HAR-RV use fixed `seed=42`; the lognormal fallback seeds from `blake2b(...)` not `hash()`;
  the dominant `empirical_non_overlapping` tier is RNG-free.
- ⇒ **The drift is real merge-drift, not noise. Re-baseline premise valid.**

## What didn't / dead ends
- **E4-E6 and the NaN/inf guards are exact no-ops** for both s35 and s27 (revert-{D20+E4E6+
  guards} is byte-identical to revert-D20 alone). The dealer multiplier degrades neutral on
  Bloomberg (no chain) so E4's PIT-T fix never bites; no degenerate inputs hit the guards.
- **The s27 ~0.2-0.4% residual after revert-D20 is a harness-provenance artifact, NOT a
  regression.** Running s27 *at its own pin SHA `0dddf76`* does NOT reproduce the pin (same
  10 tiny FAILs) — so the committed s27 pin's EV-level metrics were generated by a
  throwaway/one-off harness (the #260 F4 backtest), not the `_common.py` regression driver.
  counts / mean_realized / spearman_rho / hit_rate DO reproduce. s32/s34 (same 05-29
  re-author batch) carry the same artifact; s35's pin DOES reproduce (made by `_common`).
  Re-pinning to `_common @main` makes all four self-consistent and reproducible going forward.
